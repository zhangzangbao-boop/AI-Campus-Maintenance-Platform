# 性能测试说明

本次新增轻量、可重复执行的性能基线脚本：`scripts/perf-baseline.ps1`。脚本不启动生产外部服务，不内置账号、密码、Token 或密钥；只有在显式提供测试环境地址和测试账号时才会发起请求。

## 覆盖接口

| 场景 | 接口 | 目的 |
| --- | --- | --- |
| 登录 | `POST /api/auth/login` | 验证认证接口在并发登录下的成功率与响应时间 |
| 查询工单 | `GET /api/repair-orders/my?page=0&size=10` | 验证学生侧工单列表查询性能 |
| 创建工单 | `POST /api/repair-orders` multipart/form-data | 验证核心报修创建链路性能 |

## 指标

脚本输出 JSON，记录：

- 并发数；
- 执行次数；
- 成功数；
- 成功率；
- 平均响应时间；
- P95 响应时间；
- 首个失败原因。

## 使用方式

未提供环境参数时：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\perf-baseline.ps1
```

输出状态为 `PENDING_FULL_CHAIN_ENVIRONMENT`，表示“待全链路环境执行”，不会向任何环境发送请求。

提供测试环境后：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\perf-baseline.ps1 `
  -BaseUrl http://localhost:8080 `
  -UserId test_student `
  -Password <password> `
  -Concurrency 5 `
  -Iterations 20
```

如只希望测试登录和查询，不创建工单：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\perf-baseline.ps1 `
  -BaseUrl http://localhost:8080 `
  -UserId test_student `
  -Password <password> `
  -Concurrency 5 `
  -Iterations 20 `
  -SkipCreateTicket
```
