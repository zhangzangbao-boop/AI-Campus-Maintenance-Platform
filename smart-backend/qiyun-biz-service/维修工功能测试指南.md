本文档提供多种方式检验维修工相关功能是否可用。

## 一、单元测试（推荐）

### 运行测试类

项目已创建专门的测试类，包含维修工功能和工单流程测试：

```bash
# 在项目根目录执行
cd smart-backend/qiyun-biz-service

# 运行维修工功能测试
mvn test -Dtest=StaffFeatureTests

# 运行工单流程测试
mvn test -Dtest=TicketWorkflowTests

# 运行所有测试
mvn test
```

#### TicketWorkflowTests（工单处理流程）

| 测试方法 | 功能 | 验证点 |

## 四、常见问题排查

### 问题1：401 Unauthorized

**原因：** Token 无效或未传递

**解决：**
1. 确认登录接口返回了 token
2. 确认请求头格式正确：`Authorization: Bearer <token>`

### 问题2：403 Forbidden

**原因：** 用户角色不是 STAFF

**解决：**
1. 检查登录用户是否为维修工角色
2. 确认数据库中该用户的 `role` 字段为 `STAFF`

### 问题3：返回空列表

**原因：** 没有分配给该维修工的工单

**解决：**
1. 使用管理员账号登录
2. 通过 `/api/admin/repair-orders/{id}/assign` 接口分配工单

## 五、快速验证脚本

### 使用 curl 批量测试

```bash
#!/bin/bash

# 1. 登录获取 token
echo "=== 1. 测试维修工登录 ==="
LOGIN_RESPONSE=$(curl -s -X POST http://localhost:8070/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"userId":"staff001","password":"123456"}')

echo "$LOGIN_RESPONSE"

TOKEN=$(echo "$LOGIN_RESPONSE" | jq -r '.token')

# 2. 获取任务列表
echo -e "\n=== 2. 测试获取任务列表 ==="
curl -s -X GET "http://localhost:8070/api/tasks/my?page=0&size=10" \
  -H "Authorization: Bearer $TOKEN" | jq '.'

# 3. 获取工作台统计
echo -e "\n=== 3. 测试工作台统计 ==="
curl -s -X GET http://localhost:8070/api/staff/dashboard \
  -H "Authorization: Bearer $TOKEN" | jq '.'
```

## 六、数据库验证

直接查询数据库验证数据正确性：

```sql
-- 查看维修工的任务数量统计
SELECT 
  u.user_number,
  u.name,
  COUNT(t.id) as total_tasks,
  SUM(CASE WHEN t.status = 'IN_PROGRESS' THEN 1 ELSE 0 END) as in_progress,
  SUM(CASE WHEN t.status IN ('RESOLVED','WAITING_FEEDBACK','FEEDBACKED','CLOSED') THEN 1 ELSE 0 END) as completed
FROM sys_user u
LEFT JOIN repair_order t ON t.repairman_id = u.user_number
WHERE u.role = 'STAFF'
GROUP BY u.user_number, u.name;

-- 查看维修工的平均评分
SELECT 
  repairman_id,
  COUNT(*) as rating_count,
  AVG(rating) as avg_rating
FROM repair_feedback
GROUP BY repairman_id;
```

### 状态流转规则

| 当前状态 | 可转换状态 | 触发操作 | 操作者 |
|---------|-----------|---------|--------|
| WAITING_ACCEPT | IN_PROGRESS | 派单/接单 | 管理员/维修工 |
| WAITING_ACCEPT | REJECTED | 驳回 | 管理员 |
| IN_PROGRESS | RESOLVED | 完成工单 | 维修工 |
| IN_PROGRESS | REJECTED | 驳回 | 管理员 |
| RESOLVED | WAITING_FEEDBACK | 进入待评价 | 系统/管理员 |
| RESOLVED | CLOSED | 直接关闭 | 管理员 |
| WAITING_FEEDBACK | FEEDBACKED | 学生评价 | 学生 |
| WAITING_FEEDBACK | CLOSED | 关闭 | 管理员 |
| FEEDBACKED | CLOSED | 关闭 | 管理员 |
| REJECTED | - | 不可转换 | - |
| CLOSED | - | 不可转换 | - |

### 关键接口说明

| 接口 | 状态流转 | 说明 |
|------|---------|------|
| `POST /api/tasks/{id}/accept` | WAITING_ACCEPT → IN_PROGRESS | 维修工主动接单 |
| `PUT /api/tasks/{id}/resolve` | IN_PROGRESS → RESOLVED | 维修工完成工单 |
| `POST /api/admin/repair-orders/{id}/assign` | WAITING_ACCEPT → IN_PROGRESS | 管理员派单 |
| `PUT /api/tasks/{id}/status` | 多状态流转 | 通用状态更新 |
| `POST /api/repair-orders/{id}/evaluate` | WAITING_FEEDBACK → FEEDBACKED | 学生评价 |
