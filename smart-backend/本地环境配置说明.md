# Smart Backend 本地环境配置说明

## 重要安全提醒

**已泄露的密钥必须作废！**

本项目此前曾有 API Key、JWT Secret 等敏感信息被提交到 Git 仓库。这些密钥已被视为泄露，**必须在相关平台控制台中立即作废并重新生成**：

1. **DeepSeek API Key** - 登录 DeepSeek 平台，作废旧 Key，生成新 Key
2. **JWT Secret** - 生成新的强密钥（建议至少 256 位）
3. **数据库密码** - 如果使用的是生产环境密码，请立即修改

## 环境变量配置方法

### 方法一：Windows CMD（临时设置）

在启动服务前，在 CMD 窗口中执行：

```cmd
set NACOS_SERVER_ADDR=127.0.0.1:8848
set DB_URL=jdbc:mysql://127.0.0.1:3306/repairdb?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=UTF-8&allowPublicKeyRetrieval=true&useSSL=false
set DB_USERNAME=root
set DB_PASSWORD=你的数据库密码
set JWT_SECRET=你的JWT密钥
set AI_ENABLED=false
set DEEPSEEK_API_KEY=你的API密钥
```

**注意**：这些环境变量只在当前 CMD 窗口有效，关闭窗口后失效。

### 方法二：Windows PowerShell（临时设置）

在 PowerShell 中执行：

```powershell
$env:NACOS_SERVER_ADDR="127.0.0.1:8848"
$env:DB_URL="jdbc:mysql://127.0.0.1:3306/repairdb?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=UTF-8&allowPublicKeyRetrieval=true&useSSL=false"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="你的数据库密码"
$env:JWT_SECRET="你的JWT密钥"
$env:AI_ENABLED="false"
$env:DEEPSEEK_API_KEY="你的API密钥"
```

**注意**：这些环境变量只在当前 PowerShell 会话有效。

### 方法三：使用 .env 文件（推荐）

1. 复制项目根目录的 `.env.example` 为 `.env`
2. 编辑 `.env` 文件，填入你的实际配置值
3. `.env` 文件已被 Git 忽略，不会被提交

## 启动 Nacos

本项目使用 Nacos 作为服务发现中心。

1. 下载 Nacos：https://nacos.io
2. 解压后进入 `bin` 目录
3. Windows 单机模式启动：
   ```cmd
   startup.cmd -m standalone
   ```
4. 访问 Nacos 控制台：http://127.0.0.1:8848/nacos
5. 默认账号密码：nacos/nacos

## Smart Backend 服务启动顺序

请按以下顺序启动服务：

1. **Nacos**（服务发现中心）
   - 端口：8848
   - 必须最先启动

2. **qiyun-user-service**（用户服务）
   - 端口：9003（可通过 `USER_SERVICE_PORT` 环境变量覆盖）
   - 处理所有用户业务：认证、个人信息、用户管理
   - **必须配置与 biz-service 相同的 JWT_SECRET**

3. **qiyun-repair-service**（报修服务）
   - 端口：9004（可通过 `REPAIR_SERVICE_PORT` 环境变量覆盖）
   - 处理工单创建、维修任务、工单评价、分类等报修业务

4. **qiyun-ai-service**（AI 分析服务）
   - 端口：9002
   - 处理 `/api/ai/**` 请求

5. **qiyun-biz-service**（业务服务）
   - 端口：9001
   - 处理运维统计、通知、审计等后台管理业务

6. **qiyun-gateway**（API 网关）
   - 端口：8070
   - 所有 API 请求入口

### 启动命令示例

在每个服务的目录下执行：

```cmd
cd smart-backend\qiyun-user-service
mvn spring-boot:run

cd smart-backend\qiyun-ai-service
mvn spring-boot:run

cd smart-backend\qiyun-biz-service
mvn spring-boot:run

cd smart-backend\qiyun-gateway
mvn spring-boot:run
```

## 环境变量配置要求

### qiyun-user-service 必需配置

| 环境变量 | 说明 |
|----------|------|
| `DB_URL` | 数据库连接 URL |
| `DB_USERNAME` | 数据库用户名 |
| `DB_PASSWORD` | 数据库密码 |
| `JWT_SECRET` | JWT 签名密钥（必须与 biz-service 相同） |

### qiyun-biz-service 必需配置

| 环境变量 | 说明 |
|----------|------|
| `DB_URL` | 数据库连接 URL |
| `DB_USERNAME` | 数据库用户名 |
| `DB_PASSWORD` | 数据库密码 |
| `JWT_SECRET` | JWT 签名密钥（必须与 user-service 相同） |

### 重要：JWT_SECRET 必须一致

**qiyun-user-service 和 qiyun-biz-service 必须使用相同的 JWT_SECRET**，否则：
- user-service 签发的 Token 无法被 biz-service 验证
- 用户登录后无法访问需要认证的业务接口

## Gateway 认证路由回滚方法

如果 qiyun-user-service 启动失败，可以通过以下方式回滚：

1. 编辑 `qiyun-gateway/src/main/resources/application.yml`
2. 注释或删除以下路由配置：
   ```yaml
   # - id: user-auth-route
   # - id: admin-users-route
   # - id: users-route
   ```
3. 重启 qiyun-gateway
4. `/api/auth/**`、`/api/users/**`、`/api/admin/users/**` 请求将回落到 biz-service

## 无 AI Key 时启动

如果没有 DeepSeek API Key，可以禁用 AI 功能启动：

```cmd
set AI_ENABLED=false
```

或设置空值：

```cmd
set DEEPSEEK_API_KEY=
```

此时 AI 相关接口会返回降级响应，不影响其他业务功能。

## 安全注意事项

1. **不要把真实 .env 文件提交到 Git**
2. **不要在代码中硬编码任何密钥**
3. **定期更换 JWT Secret**
4. **生产环境使用强密码和强密钥**
5. **API Key 仅存储在本地环境变量或安全的密钥管理服务中**
6. **user-service 和 biz-service 必须使用相同 JWT_SECRET**
7. **管理员重置密码返回临时明文密码，建议后续增加首次登录修改密码机制**
